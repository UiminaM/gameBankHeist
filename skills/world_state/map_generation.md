# Skill: генерация карты

## Задача

Сгенерировать `WorldSpec` (см. Pydantic-схему) для заданного уровня сложности.

## Параметры

| difficulty | size | cameras | npcs | cone_length |
|---|---|---|---|---|
| easy | 6×6 | 4 | 5 | 3 |
| medium | 8×8 | 6 | 8 | 4 |
| hard | 10×10 | 8 | 12 | 5 |

## Инвариант

- **Минимум 2 различных пути** от единственного `entry` до `vault`.
- Все клетки `entry`, `vault`, `exit_A`, `exit_B` различны и не совпадают со стенами/NPC.
- От `vault` достижим каждый из двух выходов.

## Правило равномерного распределения

Карта делится на **четыре квадранта** (2×2 разбиение по линиям `x = size/2` и `y = size/2`).

- **Стены**: ни в одном квадранте не должно быть больше `⌈total_walls / 4 · 1.75⌉` стен.
  Для `size=8` (≈11 стен) это ≈5 стен на квадрант, не больше.
  Запрещено размешать стены в одну строку/столбец, в одну половину карты или в один угол.
- **Камеры**: в каждом квадранте не больше `⌈cameras / num_zones⌉ + 1` камер,
  где `num_zones = ⌈√cameras⌉²`. Для 5 камер это 2 на зону, для 8 — тоже 2.
  Каждая камера — в своём «куске» карты, не в одной локации.
- **NPC** должны быть рассеяны по карте, а не находиться рядом.

## Правила расстановки камер

- Камеры — на пересечениях коридоров, конусы перекрывают «узкие места».
- Конусы **не** перекрываются стеной до конца своей длины.
- Конусы могут пересекаться.
- Камеры из разных квадрантов смотрят так, чтобы их конусы охватывали разные части карты.

## Расстановка NPC

- Не на стенах, не на спец-клетках.
- Поле `archetype` это один из вариантов {"aggressive", "scared", "neutral"}.
- Направление патруля (`move_direction`) NPC выдаёт оркестратор в начале 2 фазы — в JSON его указывать не нужно.

## Формат ответа

Только валидный JSON по схеме `WorldSpec`. Пример:

```json
{
  "size": 10,
  "walls": [[0,0], [0,1]],
  "cameras": [{"id": "c1", "pos": [3,4], "direction": "S", "length": 3}],
  "npcs": [{"id": "npc1", "pos": [5,7], "archetype": "scared"}],
  "entry": [0,2],
  "vault": [7,7],
  "exits": [[0,5], [9,2]]
}
```
